## V14a.1 Row count тАФ five data rows (one per R92 field)

R92's schema is frozen with **exactly five** displayable values. The screen shows one row per value
plus a title and a back hint (the "six rows" in GDD ┬зV14.11 = **title + 5 rows**; not a sixth datum):

| # | Row (snake_case field тЖТ Writer's human label) | Source | Treatment |
|---|---|---|---|

| 3 | `enemies_killed` | R92 store | counter |
| 4 | `asteroids_destroyed` | R92 store | counter |
| 5 | `bosses_killed` | R92 store | counter |

Values are read from the R94 in-memory store and rendered as **natural integers** (`str(value)`,
**not** zero-padded тАФ unlike the HUD `SCORE {:05d}`). `highscore` is the same field GAME_OVER's BEST
line reads (V14.6) тАФ consistent, just the full-ledger placement.

## V14a.4 тШЕ Exact pixel layout тАФ LOCKED

Screen 600├Ч800, center `cx = W//2 = 300`. Title and hint are horizontally centered at `cx`. Ledger
rows are a **two-column** layout inside a centered band: **label anchored `midleft`**, **value anchored
`midright`** on the same center-y, so the value digit columns line up at the right rail.

| # | Element | Font | Anchor | Position | Color |
|---|---|---|---|---|---|
| тАФ | Title | `FONT_BIG` | `midtop` | (300, **130**) тЖТ center y 154 | `PLAYER` |

| 1 | highscore | `FONT_MID` | label `midleft` (100, cy) ┬╖ value `midright` (500, cy) | cy=**264** | label+value `TEXT` |
| тАФ | Headline divider | 1 px line | тАФ | y=**304**, x 100тЖТ500 | `STAR_FAR` |
| 2 | runs | `FONT_MID` | label `midleft` (100, cy) ┬╖ value `midright` (500, cy) | cy=**344** | label `TEXT_DIM` ┬╖ value `TEXT` |
| 3 | enemies_killed | `FONT_MID` | same | cy=**404** | label `TEXT_DIM` ┬╖ value `TEXT` |
| 4 | asteroids_destroyed | `FONT_MID` | same | cy=**464** | label `TEXT_DIM` ┬╖ value `TEXT` |
| 5 | bosses_killed | `FONT_MID` | same | cy=**524** | label `TEXT_DIM` ┬╖ value `TEXT` |
| тАФ | Back hint | `FONT_SMALL` | `midtop` | (300, **712**) тЖТ center y 721 | `TEXT_DIM` |

**Column rails & overflow budget (so labels never collide with values):**

- **Label rail:** left edge x=**100**; budget **тЙд 260 px** render width (тЙИ тЙд18 chars at `FONT_MID`).
- **Value rail:** right edge x=**500**; reserve **тЙд 120 px** (тЙИ тЙд8 digits тАФ real values never approach this).
- Worst case: label right edge тЙд 360, value left edge тЙе 380 тЖТ **тЙе 20 px guaranteed gap**. Band is
  centered (100тАж500, mid=300). Writer keeps labels inside the 260 px budget (e.g. "ASTEROIDS", "BOSSES").

**Vertical rhythm:** rows are ~24 px tall; row centers are тЙе 60 px apart (60 px pitch for counters,
80 px highscoreтЖТruns to clear the divider) тЖТ no two rows overlap. Block spans y 130тАж548; back hint at
y 712.

## V14a.5 Draw recipe тАФ `draw_stats(screen, store)` (paste-ready sketch)

```python
def draw_stats(screen, store):
    """STATS screen: title + 5-row lifetime ledger + back hint, over the scrolling
    starfield (no dim, no arc). `store` exposes the 5 R92 ints. (art_spec ┬зV14a.4)"""
    cx = C.W // 2  # 300

    # Title (cyan, like START)
    title = _FONTS["big"].render(C.STATS_TITLE, True, C.PLAYER)
    screen.blit(title, title.get_rect(midtop=(cx, C.STATS_TITLE_Y)))

    # Two divider rules
    for y in (C.STATS_DIV_HEADER_Y, C.STATS_DIV_HEADLINE_Y):
        pygame.draw.line(screen, C.STAR_FAR, (C.STATS_BAND_L, y), (C.STATS_BAND_R, y), 1)

    # Ledger rows: (label_str, value_int, label_color). Strings = Writer; field order = V14a.4.
    rows = [
        (C.STATS_LBL_HIGHSCORE, store.highscore,           C.TEXT),       # headline
        (C.STATS_LBL_RUNS,      store.runs,                C.TEXT_DIM),
        (C.STATS_LBL_ENEMIES,   store.enemies_killed,      C.TEXT_DIM),
        (C.STATS_LBL_ASTEROIDS, store.asteroids_destroyed, C.TEXT_DIM),
        (C.STATS_LBL_BOSSES,    store.bosses_killed,       C.TEXT_DIM),
    ]
    for (label, value, lbl_color), cy in zip(rows, C.STATS_ROW_CY):
        lab = _FONTS["mid"].render(label, True, lbl_color)
        val = _FONTS["mid"].render(str(value), True, C.TEXT)
        screen.blit(lab, lab.get_rect(midleft=(C.STATS_BAND_L, cy)))
        screen.blit(val, val.get_rect(midright=(C.STATS_BAND_R, cy)))

    # Back hint (Tab/Esc тАФ Back; string = Writer)
    hint = _FONTS["small"].render(C.STATS_HINT, True, C.TEXT_DIM)
    screen.blit(hint, hint.get_rect(midtop=(cx, C.STATS_HINT_Y)))
```

## V14a.7 Render order тАФ STATS in the state-overlay slot, no in-run HUD

STATS slots into the existing state-overlay dispatch (v8 ┬зV8.6 layer 11), **like START**:

1. `screen.fill(BG)`
2. Starfield Far тЖТ Mid тЖТ Near *(scrolls during STATS тАФ `update_starfield` runs in all states)*
3. **тАФ no world is drawn** (V14.2: STATS runs no world update; no asteroids/enemies/player/bullets)
4. **тАФ no in-run HUD** (score / HP bar / bomb / buff-pills are suppressed, exactly as on START тАФ
   there is no live run, so a stale score/HP must not show)
5. **State overlay (last): `STATS` тЖТ `draw_stats(screen, store)`** тАФ title, 2 rules, 5 rows, back hint.

No arc layer, no activation flash, no dim. (START/PAUSE/GAME_OVER dispatch unchanged.)
