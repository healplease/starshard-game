## V10.3 ★ Idle visibility — whole widget drawn ONLY while Q is held (R80 / GDD §V10.6.1)

This is the one deviation the Designer registered (GDD §V10.2/§V10.6.1), and it is an **art** rule, so
it is restated here as the binding contract:

| Screen | What is drawn when **Q is NOT held** (`q_hold_frames == 0`) | What is drawn while **Q is held** (`q_hold_frames > 0`) |
|---|---|---|
| **START** | **nothing** — screen visually unchanged from today | track ring (`HP_BACK`) **+** CW amber fill (`HP_AMBER`), both at `START_ARC_CENTER` |
| **GAME_OVER** | **nothing** — screen visually unchanged from today | track ring + CW amber fill at `GAMEOVER_ARC_CENTER` |
| **PAUSE** | **track ring always on** (v8 — unchanged) | track + fill (v8 — unchanged) |

- On START / GAME_OVER the **track is gated with the fill** — there is no persistent menu panel for a
  bare amber ring to belong to, so an always-on ring would read as a mystery indicator. Drawing the
  whole widget only while held makes it self-explanatory: it appears the instant you press Q and
  vanishes on release/quit. This satisfies AC68's "drawn **with and without** the arc present" — when
  not held, the START/GAME_OVER text rects are the *only* rects, so no overlap can occur; when held,
  the arc rect is proven clear below (§V10.4).
- **`draw_pause` is NOT modified.** The PAUSE overlay keeps its v8 always-on empty track (it *does*
  have a panel to anchor to). The Programmer reuses the v8 arc *geometry* for the new screens but does
  not change PAUSE behaviour.

## V10.9 Draw recipe — reusable arc helper + two guarded screen calls (paste-ready)

Factor the v8 arc body (`hud.py` ~L223–244) into one small helper that draws the v8 visual at any
centre, then call it from two tiny screen helpers that draw **only while held**. `draw_pause` is left
exactly as shipped.

```python
def draw_quit_arc(screen, center, q_hold_frames):
    """The v8 Q-hold progress arc (track ring + CW amber fill) at an arbitrary centre.
    Visual is identical to draw_pause's arc (art_spec v8 §V8.4): r=22, stroke=3,
    HP_AMBER fill / HP_BACK track, CW from 12 o'clock. (art_spec v10 §V10.2)"""
    cx, cy = center
    r = C.PAUSE_ARC_R                                    # 22
    rect = pygame.Rect(cx - r, cy - r, 2 * r, 2 * r)     # 44×44 bounding box
    # Track (empty ring) — full circle
    pygame.draw.arc(screen, C.HP_BACK, rect, 0, 2 * math.pi, C.PAUSE_ARC_STROKE)
    # Fill arc — CW from 12 o'clock: end = π/2, start = π/2 − fill·2π
    fill = q_hold_frames / C.PAUSE_QUIT_FRAMES          # 0.0 → 1.0
    if fill > 0:
        end_a = math.pi / 2
        start_a = end_a - fill * 2 * math.pi
        pygame.draw.arc(screen, C.HP_AMBER, rect, start_a, end_a, C.PAUSE_ARC_STROKE)


def draw_start_quit_arc(screen, q_hold_frames):
    """START: draw the whole widget ONLY while Q is held (art_spec v10 §V10.3)."""
    if q_hold_frames > 0:
        draw_quit_arc(screen, C.START_ARC_CENTER, q_hold_frames)


def draw_gameover_quit_arc(screen, q_hold_frames):
    """GAME_OVER: draw the whole widget ONLY while Q is held (art_spec v10 §V10.3)."""
    if q_hold_frames > 0:
        draw_quit_arc(screen, C.GAMEOVER_ARC_CENTER, q_hold_frames)
```

In `app._draw()`: after `hud.draw_start(...)` call `hud.draw_start_quit_arc(screen, self.q_hold_frames)`;
after `hud.draw_gameover(...)` call `hud.draw_gameover_quit_arc(screen, self.q_hold_frames)`. PAUSE's
`hud.draw_pause(...)` call is unchanged.

> **Note on factoring `draw_pause`:** optional. The Programmer *may* have `draw_pause` call
> `draw_quit_arc` for its fill (to dedupe), but PAUSE's **always-on track** must remain, so the
> behaviour of `draw_pause` does not change. Per the v10 art task, leaving `draw_pause` entirely as-is
> is the safe default — the new screens use the helper above; PAUSE is untouched.
