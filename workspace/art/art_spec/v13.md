## V13.6 Draw recipe — pass a fill colour into the shared helper

Generalise `draw_hold_arc(...)` to take the fill colour (default keeps Q amber), so the Q calls are
unchanged and the R calls pass violet:

```python
def draw_hold_arc(screen, center, hold_frames, threshold, fill_color=C.HP_AMBER):
    """v8 hold-progress arc at `center`: HP_BACK track + CW fill from 12 o'clock.
    fill_color defaults to HP_AMBER (Q arc); R arc passes BONUS_BOMB. (art_spec v13 §V13.3)"""
    cx, cy = center
    r = C.PAUSE_ARC_R
    rect = pygame.Rect(cx - r, cy - r, 2 * r, 2 * r)
    pygame.draw.arc(screen, C.HP_BACK, rect, 0, 2 * math.pi, C.PAUSE_ARC_STROKE)  # track
    fill = hold_frames / threshold
    if fill > 0:
        end_a = math.pi / 2
        start_a = end_a - fill * 2 * math.pi
        pygame.draw.arc(screen, fill_color, rect, start_a, end_a, C.PAUSE_ARC_STROKE)
```

The two R-arc helpers (v12 §V12.9) pass `BONUS_BOMB` and the v13 centres; **draw them after the Q
arcs** so violet wins on overlap:

def draw_pause_restart_arc(screen, r_hold_frames):
