## V20s.3 Display name for EVERY lethal source + the unknown FALLBACK (R131 — the full table)

Every source that can deal the killing blow maps to a display name; a projectile/beam kill resolves
via `source`→owner to that owning ship's name (the Programmer's mapping, GDD §V20.5). All are
**all-caps** (they read as the subject of "Killed by ___" and match the blessed-name voice). Paste
verbatim:

| Lethal source (what hit you) | Display name (paste verbatim) | Notes |
|---|---|---|
| Asteroid **or** debris (body) | `ASTEROID` | one name covers both — debris is just small asteroid rock (v1 §4); "ASTEROID" reads true for either, no need to split |
| REGULAR enemy (body or its RED bullet) | `REGULAR` | = the v5 kind name verbatim |
| HEAVY enemy (body or its GREEN/RED pellet) | `HEAVY` | = the v5 kind name verbatim |
| SCOUT enemy (body or its CYAN bullet) | `SCOUT` | = the v5 kind name verbatim |
| MOTHERSHIP boss (body or its bullets) | `MOTHERSHIP` | = the v7 boss name verbatim |
| NOVA boss (body or its bullets) | `NOVA` | = the v16 boss name verbatim |
| LASER enemy (body or its **beam**) | `LASER` | = the v20 kind name verbatim (§V20s.1) — covers a beam kill via `source`→owner |
| **Unknown / undeterminable** (FALLBACK) | `SOMETHING` | R131 fallback — used iff the source can't be resolved (culled before read, future un-named source); never blank/raw-ID/crash |

**Why these exact names:**
- **Reuse the already-blessed names verbatim** (REGULAR/HEAVY/SCOUT/MOTHERSHIP/NOVA/LASER) — one
  vocabulary; the thing that killed you is named the same on the death line as everywhere else. No new
  words to learn, nothing to keep in sync (the name *is* the kind key / boss name).
- **`ASTEROID` for both rock hazards** — the game's only non-ship lethal body is the asteroid/debris
  family (v1 §4); "ASTEROID" is true for a big rock or a small chip, and reads better than a vague
  "ROCK" or an awkward "DEBRIS" for the small ones. One name, both bodies.
- **`SOMETHING` as the fallback** — terse, on-voice arcade, and grammatical in the sentence
  ("Killed by SOMETHING"). It's the never-blank guarantee R131 demands: any source the tracker can't
  resolve still prints a clean, defined line instead of a blank, a raw ID, or a crash. All-caps so it
  sits in the line like a name; it should appear **only** as a true safety net, never in normal play.

> **No article in the format.** `f"Killed by {name}"` takes the bare name with no "a/an/the" — so one
> format string serves every source uniformly (the Programmer never branches on article). "Killed by
> ASTEROID" / "Killed by LASER" read as arcade-terse name tags, consistent with the all-caps voice; an
> article would force per-source grammar for zero benefit. **Decided: bare name, no article.**

## V20s.4 Laser windup / telegraph warning copy — **NONE needed (explicit decision)**

**Decision: v20 adds NO windup/telegraph warning *copy*.** Stated plainly so the Programmer doesn't
hunt for a string that doesn't exist and QA knows the text surface gains *only* the §V20s.2 line.

Why no warning text is warranted (each checked against the specs):
1. **The telegraph is already entirely *visual*.** The whole "read the tell and walk out" skill (GDD
   §V20.2) is carried by the Artist's windup tell — the searing-orange **charging ring** on the eye +
   the faint thin **aim-line** (art_spec §V20a.2/§V20a.3.1). That *is* the warning; a printed word would
   be redundant and slower to read than the line pointing at you.
2. **No per-enemy on-screen text exists to attach it to.** Like the v5 enemies (story §V5.2), the LASER
   is identified visually, not by a HUD label/caption — there is no enemy-text surface, and adding a
   floating "WARNING" over a *normal-stream enemy* would be screen clutter on a single-screen cabinet
   (the boss `WARNING` klaxon is a once-per-fight event, not a per-enemy-shot label — a different beast).
3. **It would fight the design.** A scary printed "WARNING" on every laser windup (which fires
   repeatedly, GDD §V20.3.4) would nag constantly and undercut the *quiet, readable, periodic* threat
   the Designer wants (beam, breathe, beam). The visual tell is the right register.

**Net:** the only new on-screen strings in v20 are the **`LASER`** name (also used as an attribution
display name) and the **`Killed by <name>`** line + its source-name table/fallback. The Programmer
pastes nothing else from the Writer this increment; all v1–v19 copy still applies verbatim.
