## V8.1 PAUSE_TITLE — **"PAUSED"**

The GDD (§V8.6.1) and art_spec (§V8.3.2 row 1) both use "PAUSED" as the heading throughout.
I **bless it verbatim** — all-caps, no punctuation, no synonym ("ON HOLD", "GAME PAUSED", etc.).

Rationale:
- **Arcade voice is terse.** "PAUSED" is the genre standard — immediately readable at a glance,
  identical to the mental model. No word wastes space or adds ambiguity.
- **All-caps matches every other heading** (`STARSHARD`, `GAME OVER`, `MOTHERSHIP`, `WARNING`).
- **Color distinguishes the state** — the heading is rendered `PLAYER` cyan `#50DCFF`, visually
  opposite to GAME_OVER's `HP_RED` heading. The text itself need not do extra state-signalling work.
- **Length is safe.** "PAUSED" (6 chars) at `FONT_BIG` 64 is comfortably inside the 600 px window.

## V8.2 Hint lines — three strings, all `FONT_SMALL` / `TEXT_DIM`, centered at x = 300

The art_spec §V8.3.2 names exactly three hint slots in order: (1) Resume, (2) Q-hold quit,
(3) Restart. Each renders as a single `FONT_SMALL` (size 22 → ~18 px tall) line in `TEXT_DIM`
color `(140, 148, 170)`, horizontally centered at `x = W//2 = 300` in a 600 px window.

**Key-first format** (matches `GAMEOVER_KEYS` discipline: key on left, action label on right)
keeps all three lines scannable in a vertical column — the eye reads down the left side for keys,
right side for labels.

### V8.2.1 PAUSE_HINT_RESUME — `"Esc  Resume"`

The resume action is a second Esc press (GDD §V8.2.2 table: PAUSE + K_ESCAPE → PLAY).

- **11 chars** at `FONT_SMALL` 22 ≈ **~121 px** (≪ 600 px; single-line safe). ✓
- "Esc" is the canonical key name across the whole control vocabulary (CONTROLS_1/2 on START).
- "Resume" is the plainest, least ambiguous English word for this action — not "Unpause",
  not "Continue", not "Return" (each imports a different register).

### V8.2.2 PAUSE_HINT_QUIT — `"Hold Q  Quit"`

The quit action is Q held for 30 frames (GDD §V8.3: `PAUSE_QUIT_FRAMES = 30`, 0.5 s).

- **12 chars** at `FONT_SMALL` 22 ≈ **~132 px**. ✓
- "Hold Q" describes the gesture (not just the key), signalling to the player that a tap will
  not quit — deliberate resistance is required. This matches the arc visual feedback:
  the player sees the arc sweeping and reads "Hold Q" confirming they are doing the right thing.
- "Quit" (not "Exit") is the direct, arcade-standard action label — single syllable, no ambiguity.
- "Hold" capitalised at line start (key-first format): "Hold Q" reads as a verb phrase giving
  the instruction, then "Quit" names the outcome. Parallel to "Esc  Resume" and "R  Restart".

### V8.2.3 PAUSE_HINT_RESTART — `"R  Restart"`

The restart action is R pressed from PAUSE (GDD §V8.7: `pygame.K_r` → `reset_run()` + PLAY).

- **10 chars** at `FONT_SMALL` 22 ≈ **~110 px**. ✓
- Identical format to the GAME_OVER key ("R  Restart" half of the old `GAMEOVER_KEYS`) —
  intentional: **R always means restart**, regardless of state. One mental model, two screens.
- No "from pause" qualification needed — context is obvious from the overlay.

## V8.3 CONTROLS_2 rewrite — `"Esc  Pause · hold Q  Quit"` ⚠ REWRITE

> **Old value** (`config.py` line 302): `"QUIT  Esc"`
> **New value**: `"Esc  Pause · hold Q  Quit"`

**Why the old string is now wrong:** Esc no longer quits. The BA ruling (GDD §V8.2.2 / R73)
removes the `K_ESCAPE → exit` shortcut entirely. A START screen that still says "QUIT  Esc" will
actively mislead the player on v8.

**What the new string conveys:**
- `"Esc  Pause"` — Esc now pauses the game mid-run (the primary, always-reachable action).
- `"hold Q  Quit"` — quitting requires the deliberate 0.5 s hold, from inside the pause overlay.
- The `·` bullet separator (same as `CONTROLS_1`'s `·`) cleanly divides the two bindings on one
  line.

**Length check:** "Esc  Pause · hold Q  Quit" is **26 chars**. At `FONT_SMALL` size 22,
~11 px/char average → **~286 px** (< 600 px; single-line safe). ✓

## V8.4 GAMEOVER_KEYS rewrite — `"R  Restart"` ⚠ REWRITE

> **Old value** (`config.py` line 305): `"R  Restart      Esc  Quit"`
> **New value**: `"R  Restart"`

**Why the Esc clause is removed:** In GAME_OVER, Esc is **silently ignored** (GDD §V8.2.3 —
no transition, no log, no feedback). Displaying "Esc  Quit" in GAME_OVER would be a lie: the key
does nothing. Drop the clause entirely rather than substitute a misleading alternative.

**Why not add a note about Q-hold?** Q-hold operates only inside the PAUSE overlay (GDD §V8.3 —
"Q-hold logic runs only when `state is GameState.PAUSE`"). GAME_OVER is not PAUSE; holding Q from
GAME_OVER does nothing. Mentioning it here would confuse more than help.

**Result:** `"R  Restart"` — the one thing the GAME_OVER screen can actually do. Clean, honest.
