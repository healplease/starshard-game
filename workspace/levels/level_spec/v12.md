## §V12.2 Edge case: the second hold counter is UI-only

v12 adds a **second** hold counter, `App.r_hold_frames`, beside the existing `q_hold_frames`. Both
are **UI hold counters, not game clocks** — neither is consulted by any spawner, ramp formula, or
boss breakpoint. The §V12.4 spine resets **both** to 0 on every state transition; that reset touches
UI state only and never reads or writes `w.frame`, the ramp `t`, or any boss timer. PAUSE freezes
the run clock (v8 §V8.1) and GAME_OVER doesn't advance it, so winding up an R-hold on either screen
buys zero difficulty discount and costs zero economy — exactly as the Q-hold does today.

`reset_run()` semantics are **unchanged** (GDD §V12.5 #3): restart is behaviourally identical to the
shipped PAUSE (v8 R74) and GAME_OVER (v1 R13) restart — it just arms over a hold instead of an
instant press. The economy starting state on a fresh run (charges → 2, etc.) is therefore identical.
