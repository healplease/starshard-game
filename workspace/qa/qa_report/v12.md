## AC69–AC77 — verified
- **AC69 — hold R on PAUSE → restart; tap = no-op.** Harness `v12/AC69` drives the REAL loop:
  Esc→PAUSE then R held f4..f33 (30 f) → `reset_run()` → PLAY, **both** counters 0. Harness `v8/R74`
  confirms a single R **tap** no longer restarts (KEYDOWN branch removed, §V12.5). ✅
- **AC70 — hold R on GAME_OVER → restart; tap = no-op.** My **P1** holds R 30 f from GAME_OVER in the
  real loop → PLAY (q=0, r=0). My **P5** confirms a 29-f hold does **not** restart and a release at 29 f
  snaps the counter to 0. ✅
- **AC71 — R arc visible, non-overlapping.** Harness `v12/AC71` rect check: PAUSE R arc `(200,483)`
  and GAME_OVER R arc `(200,545)` overlap **neither** their screen's Q arc (`(300,483)`/`(300,545)`)
  **nor** any title/hint/score/key text rect, and stay inside the window. Render-smoke (AC77) draws
  both arcs with no exception. ✅
- **AC72 — Q and R counters independent.** Harness `v12/AC72`: holding R advances only `r_hold_frames`
  (q stays 0); holding Q leaves `r_hold_frames` at 0. My **P3** adds the both-held case: Q+R held
  together advance **both** counters in lock-step (q=11, r=11 at f10), independently. My **P4**: holding
  R for 45 f restarts but **never** trips `quit_via_qhold`. ✅
- **AC73 — both counters reset on every transition; never pre-filled (THE #1 RISK).** Harness
  `v10/AC66` re-checks all six transitions zero **both** counters (incl. #4 die-with-Q-AND-R-held →
  `both_zero`, and #5/#6 R-hold restart). My **P2** is the deeper independent probe: force a
  deterministic death (1 HP + lethal asteroid) through the real loop with R held the **whole** time →
  **no instant restart** (counter is 1, not 30, on the death frame), the GAME_OVER counter builds a
  **fresh** `1,2,3,4…`, and the restart fires only after a **full fresh 30-increment** hold. This is the
  exact, intended mirror of v10's die-with-Q-held → auto-quit-after-0.5 s (shipped + QA-passed). ✅
- **AC74 — release R before threshold cancels, no accumulation.** Harness `v12/AC74` (hold 15 f →
  release → counter snaps to 0, still GAME_OVER). My **P5** independently confirms the same on the real
  loop at the 29-f boundary. ✅
- **AC75 — hints teach the hold; no stale instant-restart copy; Q hints remain.** `PAUSE_HINT_RESTART =
  "Hold R  Restart"` and `GAMEOVER_KEYS = "Hold R  Restart      Hold Q  Quit"` — both teach **Hold R**,
  the "Hold Q  Quit" clause is preserved, and there is no bare instant "R  Restart" (every restart
  clause is prefixed `Hold `). The `v9/AC47w` string-widths gate confirms both fit ≤ W and all glyphs
  exist. ✅
- **AC76 — inactive in START / PLAY.** Harness `v12/AC76` emulates the `state in (PAUSE, GAME_OVER)`
  guard for START + PLAY → R-hold is a no-op (counter stays 0). My **negative control P6** independently
  proves R held the whole PLAY run leaves `r_hold_frames == 0` (the gesture never arms mid-run). ✅
- **AC77 — smoke 0/120 f; render-smoke both arcs; no AC1–AC68 regression.** Smoke exits 0 / 120 f with
  no scripted-input change; harness `v12/AC77` render-smoke draws PAUSE + GAME_OVER in all four hold
  combinations (neither / Q only / R only / both) with no exception; harness 65/65 (no AC1–AC68
  regression). ✅
