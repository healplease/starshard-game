## AC78–AC85 — verified
- **AC78 fresh install.** Harness `v14/AC78`: missing file → all five = 0, `load()` does **not** create
  the file; first `save()` writes exactly the six keys (`version` + 5), `version==1`. ✅
- **AC79 round-trip.** Harness `v14/AC79`: saved store reloads byte-equal in a fresh `load()`. ✅
- **AC80 corrupt/partial.** Harness `v14/AC80`: unparseable garbage → zeros; non-object root (array) →
  zeros; partial file recovers good fields and defaults negative/non-int/missing to 0; **unknown
  `version` distrusts the whole shape → zeros**. Never raises. ✅
- **AC81 flush-only-on-two-triggers.** Harness `v14/AC81`: hold-R restart does **not** write; GAME_OVER
  does. **My independent probe** adds the explicit case the harness omits: an Esc→PAUSE→Esc→PLAY
  cycle writes **no file**. Code read confirms only two `save.save` call sites exist
  (`_step_play` GAME_OVER, main-loop Q-hold quit) — no per-event/timer/pause/restart write. ✅
- **AC82 count accuracy.** Independent code read confirms counters sit **exactly at the award sites**:
  `combat.py` step 1 `a.hits<=0` (`asteroids_destroyed += 1`, 1 per rock incl. large=1), step 2
  `e.hp<=0` (`enemies_killed += 1`, incl. boss minions), `encounter.on_defeat` (`bosses_killed += 1`,
  never enemies). Bomb/arrival clears + ram-consumes have **no award site** → never count; GREEN→RED
  split children are EnemyBullets (never flow through the enemy-destroy site) → never count. Harness
  `v14/AC82` exercises all of these on the real pipeline. ✅
- **AC82r runs semantics.** Harness `v14/AC82r`: `runs` increments once on initial START→PLAY and once
  per hold-R restart; **not** on Esc-resume. Code read confirms exactly two `store.runs += 1` sites
  (START→PLAY non-Q/non-Tab branch, `_restart_hold_step`). A quit/restarted-mid-run still counts because
  it was counted at its start (the increment is in-memory, persisted at the next flush). ✅
- **AC83 highscore.** Harness `v14/AC83`: `record_highscore` is `max`, never decreases; a Score×2-boosted
  `world.score` is captured (multiplier baked in before the flush reads it). **My idempotency probe**
  confirms a GAME_OVER flush followed by a quit-from-GAME_OVER flush writes the **same** store — no
  double-count (counting is decoupled from writing). ✅
- **AC84 STATS renders.** Harness `v14/AC84`: `draw_stats` with a fully-populated store raises nothing
  and all title/5-row/hint rects are mutually non-overlapping (label rail x100 / value rail x500). ✅
- **AC85 headless-safe I/O.** See Gates above + harness `v14/AC85` (smoke App resolves a temp path, not
  the real one; `resolve_path(headless=True)` with no env falls back to a temp `.json`). ✅
