### Notes / observations (no action required)
- The single-aim-routine design is the clean regression guarantee: `make_enemy_bullet` adds `uniform(−cone,
  +cone)` to the v1 base aim, so cone 0 is provably identical to the pre-v5 shot (AC29) and all three kinds
  share one code path. The GREEN pellet freezes `u` + `S` at fire time only; nothing reads the player again
  (R39.2) — verified by the teleport test.
- The split is implemented as a **frozen timer** (`round(S/speed)`) decremented in `physics`; maturation is
  gated on the pellet still being on-screen, so the R39 edge cases (off-screen / player-hit before split →
  no children) fall out of the existing despawn/combat filters with no special-casing.
- Collision stays a flat r=5 for every enemy-bullet family (RED/GREEN/CYAN) — the GREEN pellet is only
  *drawn* at r=8 — so the v2 collision pipeline is untouched and children are literally ordinary red bullets.
